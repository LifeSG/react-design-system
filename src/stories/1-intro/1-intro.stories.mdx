conform to the LifeSG design language with ease.

## Usage

### 1. Setup

#### Specifying the theme (Recommended)

The design system has a concept of themes that governs the style of the components. To specify the theme, you will need the `ThemeProvider` from
[styled-components](https://styled-components.com/). The `ThemeProvider` allows you to specify the theme that you would like to apply to the
components.

We recommend adding the `ThemeProvider` at the highest level in your app. Any children of the `ThemeProvider` will be able to access the theme.
You can read more about the `ThemeProvider` [here](https://styled-components.com/docs/advanced#theming).

```tsx
/** App.tsx **/

import React from "react";
import { ThemeProvider } from "styled-components";
import { BaseTheme } from "react-lifesg-design-system";

const App = () => {
    return (
        <ThemeProvider theme={BaseTheme}>
            {/* Rendering logic of app */}
        </ThemeProvider>
    );
};

export default App;
```

#### Using without theme

You can also use the design system without specifying the theme. But the rendered components will use the default theme instead.

### 2. Using the components

```tsx
/** SomePage.tsx **/

import React from "react";
import { Button, Text } from "react-lifesg-design-system";
import { Container } from "./SomePage.styles";

const SomePage = () => {
    const doSomething = () => {
        // Perform an action
    };

    return (
        <Container>
            <Text.H2>Subscribe here</Text.H2>
            <Button.Default onClick={doSomething}>Subscribe</Button.Default>
        </Container>
    );
};

export default SomePage;
```
