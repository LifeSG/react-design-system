# Media Query

To target different media widths, you can make use of the `MediaQuery` helper in the design system.

Here are the list of media width breakpoints that the design system is using

| Name        |   Size Ranges   | Breakpoint value |
| ----------- | :-------------: | :--------------: |
| `mobileS`   |    <= 320px     |      320px       |
| `mobileM`   |  321px - 375px  |      375px       |
| `mobileL`   |  376px - 480px  |      480px       |
| `tablet`    | 481px - 1199px  |      1199px      |
| `desktopM`  | 1200px - 1380px |      1399px      |
| `desktopL`  | 1400px - 1999px |      1999px      |
| `desktop4k` |    >= 2000px    |      3840px      |

> **Note**, as of `v6.0.0-alpha.63`, `desktopS` will no longer be available as it has been subsumed by `tablet`.

## Usage

You can either target styles using the `max-width` or `min-width` breakpoints. Depending on your approach towards a desktop-first or mobile-first implementation.

### Max Width

```tsx
import { MediaQuery } from "react-lifesg-design-system";

const Components = styled.div`
	${MediaQuery.MaxWidth.mobileL} {
		// Your style
	}
`;

// Resolves to
@media screen and (max-width: 480px) {

}
```

### Min Width

```tsx
import { MediaQuery } from "react-lifesg-design-system";

const Components = styled.div`
	${MediaQuery.MinWidth.mobileL} {
		// Your style
	}
`;

// Resolves to
@media screen and (min-width: 480px) {

}
```
