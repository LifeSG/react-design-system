# The basics

## Configuration

To configure the theme in your project,

1. Wrap your components in the `ThemeProvider` context; typically this is done at the root of the application
2. Specify the theme via the `theme` prop
3. Specify the mode (light or dark) via the `themeMode` prop
4. Import the theme-specific stylesheet

```tsx
import { ThemeProvider } from "@lifesg/react-design-system/theme";

import "@lifesg/react-design-system/theme/styles/lifesg.css";

const App = () => {
    return (
        <ThemeProvider theme="lifesg" themeMode="light">
            <Component />
        </ThemeProvider>
    );
};

export default App;
```

## How it works

The theme-specific stylesheet declares design tokens as CSS varables.

Here's an snippet of LifeSG's to illustrate:

```css
[data-fds-theme="lifesg"] {
    --fds-colour-primary-50: #1768be;
    --fds-colour-text-primary: var(--fds-colour-primary-50);
    --fds-font-family: "Open Sans";
}

[data-fds-theme="lifesg"][data-fds-theme-mode="dark"] {
    --fds-colour-text-primary: var(--fds-colour-primary-60);
}
```

The `ThemeProvider` sets the appropriate `data-fds-theme` and
`data-fds-theme-mode` attributes on the root of your component tree, which
allows the CSS variables to be applied correctly.

The CSS variable names are also available as JavaScript strings, so you don't
have to hardcode them in your code.

```tsx
import { css } from "@linaria/core";
import { Colour } from "@lifesg/react-design-system/theme";

const wrapper = css`
    background: ${Colour["bg-primary"]};
`;
```

## General usage

### With CSS-in-JS

#### Linaria

```tsx
import { Colour } from "@lifesg/react-design-system/theme";
import { css } from "@linaria/core";

const component = css`
    background: ${Colour["background-error"]};
`;
```

#### Styled Components

```tsx
import { Colour } from "@lifesg/react-design-system/theme";
import styled from "styled-components";

const SomeComponent = styled.div`
    background: ${Colour["background-error"]};
`;
```

### As a constant

If you require the token as a constant value, you could retrieve the value at
runtime with the `useDesignToken` hook.

```tsx
import { Colour, useDesignToken } from "@lifesg/react-design-system/theme";

const Component = () => {
    const colour = useDesignToken(Colour["text"]);

    // ...
};
```

Additionally, you can convert px or rem-based tokens to a numeric value with the
`parsePxOrRemValue` utility.

```tsx
import {
    Breakpoint,
    parsePxOrRemValue,
    useDesignToken,
} from "@lifesg/react-design-system/theme";

const Component = () => {
    const breakpoint = parsePxOrRemValue(
        useDesignToken(Breakpoint["lg-max"]) || ""
    );

    // ...
};
```

## Dark mode

Each theme has light mode and dark mode variants. The `ThemeProvider` has
automatic switching between light and dark modes based on the user's system
preferences by default, or with `themeMode="auto"`.

You can also explicitly control the mode.

```tsx
// When you have a toggle switch in your UI
const [userPreference, setUserPreference] = useState<ThemeMode>("auto");

<ThemeProvider theme="lifesg" themeMode={themeMode}>
    <Component />
</ThemeProvider>;
```
